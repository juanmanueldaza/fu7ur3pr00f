## Summary of Changes

### suggest-next-fail-fast
Removed 5 heuristic fallback lines from `suggest_next_node`'s except block.
On LLM failure, node now returns `suggested_next: []` instead of constructing
strings from `action_items`, `gaps`, `open_questions`. Extracted `suggest_next_node`
to module-level for testability.

### remove-direct-profile-answer
Deleted `_direct_profile_answer` (50-line pattern-matching function) and its
bypass block from `execute_inner_node`. Factual queries now reach the inner graph
and coach normally — the coach's `get_user_profile` tool handles profile questions
correctly without a hard-coded fast path.

### graph-compile-once
`ConversationEngine` now compiles the LangGraph `StateGraph` once in `__init__`
instead of per-`invoke_turn` call. Callbacks wired via a mutable `_callbacks` dict
closed over at compile time and updated before each `graph.invoke()`. Eliminates
repeated graph compilation overhead.

### provider-failover
`ModelSelectionManager.get_model()` now walks the full provider chain instead of
always using `chain[0]`. Skips unconfigured providers (via `_is_configured()` +
`settings.has_*`), catches `_create_model` exceptions, and falls through to the
next config. Raises `RuntimeError` only when the chain is exhausted.
