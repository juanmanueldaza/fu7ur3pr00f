# Repository Guidelines

## Project Structure & Module Organization
- `src/fu7ur3pr00f/` is the main package. Key areas include `agents/` (core agent + tools), `chat/` (CLI UI and HITL loop), `gatherers/`, `generators/`, `llm/`, `memory/`, `mcp/`, `prompts/`, `services/`, and `utils/`.
- Tests live in `tests/` with shared fixtures in `tests/conftest.py`.
- Example config is in `.env.example`. Runtime settings are written to `~/.fu7ur3pr00f/.env`.

## Build, Test, and Development Commands
```bash
pip install -e .
pip install pyright pytest ruff

fu7ur3pr00f                 # Launch chat client
fu7ur3pr00f --debug         # Verbose logs

pytest tests/ -q            # Unit tests
pyright src/fu7ur3pr00f     # Type checking
ruff check .                # Lint
ruff check . --fix          # Auto-fix lint
```

## Coding Style & Naming Conventions
- Python 3.13 with type hints.
- Line length: 100 (ruff configured in `pyproject.toml`).
- Prefer `collections.abc` types (`Mapping`, `Sequence`) over `typing`.
- Raise exceptions instead of returning error dicts; use `ServiceError`/`NoDataError`/`AnalysisError`.
- Use dependency injection for services to keep tests isolated.

## Testing Guidelines
- Framework: `pytest` with fixtures in `tests/conftest.py`.
- Mock external services (LLMs, HTTP, ChromaDB). No real API calls in unit tests.
- Run the full suite with `pytest tests/ -q`.
- For fresh-install connectivity checks (pipx + MCP/LLM), run `scripts/fresh_install_check.sh --source local --config-from .env`.

## Commit & Pull Request Guidelines
- Conventional commits with a concise subject; add a short body when the why or impact is non-obvious.
- Never add AI attribution lines (no `Co-Authored-By` or “Generated by …”).
- PRs should include: purpose, scope, and test evidence (e.g., `pytest tests/ -q`). Add screenshots for CLI UI changes.

## Security & Configuration Tips
- PII is anonymized before sending data to LLMs; avoid bypassing `utils/security.py`.
- Portfolio fetchers enforce SSRF protections; do not relax private IP checks.
- Secrets go through `/setup` and are stored in `~/.fu7ur3pr00f/.env` with `0o600` permissions.
